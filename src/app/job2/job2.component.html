Aplios / Analista de Sistemas

Situación: Mejorar la escalabilidad y eficiencia del desarrollo de servicios críticos de ámbito financiero para el cliente Equifax mediante la creación de microservicios robustos y soluciones automatizadas que cumplieran con los estándares de calidad exigidos por el sector financiero.

Tarea: Diseñar y desarrollar microservicios robustos siguiendo patrones arquitectónicos, buenas prácticas de clean code e implementando pruebas automatizadas para garantizar la calidad del código además de contribuir al desarrollo full-stack con prácticas de desarrollo ágil.

Acción:

    • Desarrollé microservicios modulares en Java utilizando Spring Boot, implementando el patrón Repository para abstraer el acceso a la base de datos y facilitar la gestion de entidades, y el patrón MVC para estructurar el código en capas (Modelo, Vista, Controlador), separando la lógica de negocio de la presentación. Además de aplicar de un patrón indispensable para garantizar la separación de responsabilidades y mantenibilidad como lo es la Arquitectura Hexagonal.
    • Implementé APIs REST escalables y almacenamiento en caché con Redis para optimizar el rendimiento de consultas frecuentes.
    • Diseñé microservicios orquestadores funcionales que mejoraron la modularidad del sistema, optimizando la reutilización de componentes.
    • Desarrollé una aplicación en Java y Angular generadora de plantillas para servicios .NET, acelerando los despliegues mediante la automatización de tareas repetitivas. Esto redujo el tiempo de desarrollo de nuevas funcionalidades de 2 semanas a 5 días.
    • Desarrollé funcionalidades completas de consumo de APIs REST utilizando servicios y modelos Angular, asegurando una integración fluida con microservicios backend desarrollados en Spring Boot además de implementar análisis del código con SonarQube, mejorando la calidad de código y alcanzando una cobertura de pruebas superior al 90%.
    • Automaticé procesos de integración y despliegue con Jenkins y gestioné tareas con Jira, participando activamente en metodologías Scrum asegurando entregas incrementales y alineadas con el objetivo del negocio.
    • Usé GitHub y herramientas colaborativas de Google para integración continua y control de versiones.

Resultado:

    • Mejora del Rendimiento: Optimización del rendimiento del sistema mediante el uso de caché (Redis) y patrones avanzados, reduciendo los tiempos de respuesta en un 40%.
    • Reducción en Tiempos de Desarrollo: La creación de componentes reutilizables y la automatización de procesos redujo el tiempo de desarrollo de nuevas funcionalidades de 2 semanas a 5 días.
    • Colaboración Efectiva: La implementación de metodologías agiles y herramientas colaboraivas mejoró la comunicación del equipo, reduciendo retrasos en entregas en un 30%.
